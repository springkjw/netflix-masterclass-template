# 2시간 라이브코딩 스크립트

## 0-10분: 도입

- 오늘 결과물 데모 시연
- 구현 범위와 평가 기준 안내

## 10-25분: 템플릿 이해

- 폴더 구조 설명 (`app`, `components`, `services`, `constants`, `utils`)
- 타입(`Content`) 먼저 정의

## 25-60분: 메인 화면 구현

- `Header` 구현
- `HeroBanner` 구현
- `MovieRow` + `MovieCard` 구현

## 60-90분: 검색/상세

- `/search?q=` 파라미터 처리
- `/detail/:id` 상세 페이지 구현

## 90-105분: 리팩토링 포인트

- mock/api 이중화(Repository 패턴) 설명
- `any` 제거와 타입 안정성 강조
- 컴포넌트 책임 분리 점검

## 105-120분: 배포 흐름

- Dockerfile 설명
- AWS EC2 직접 배포 체크리스트 공유

## 강의 멘트 예시

- "지금은 빨리 만들기보다 나중에 수정 가능한 구조가 중요합니다."
- "URL도 상태입니다. 주소를 공유하면 같은 화면이 열려야 합니다."
- "재사용은 복붙 방지가 아니라 변경 비용 절감입니다."
